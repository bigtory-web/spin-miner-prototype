# Spin Miner - Live Game Analysis & Break System Prototype

> HTML5 Canvas 기반 단일 파일 게임 프로토타입  
> 포트폴리오 목적: 출시된 하이브리드 캐주얼 게임을 분석하고, 개선 가설을 플레이 가능한 시스템 프로토타입으로 검증

---

## 프로젝트 한 줄 소개

**Spin Miner - Break System Prototype**은 Supercent의 출시작 **Spin Miner**를 참고해, 채굴 기반 하이브리드 캐주얼 게임에 추가할 수 있는 대형 광물 공략 시스템을 설계하고 검증한 개인 포트폴리오 프로젝트입니다.

이 프로젝트는 원작을 복제하려는 목적이 아니라, 공개된 게임 구조와 플레이 경험을 바탕으로 **채굴 손맛**, **대형 오브젝트 공략**, **카고 귀환 루프**, **챕터 진행 구조**가 어떻게 확장될 수 있는지 검증하기 위한 시스템 제안형 프로토타입입니다.

## 프로젝트 접근 방식

이 포트폴리오는 단순히 "비슷한 게임을 만들어봤다"가 아니라, 아래 흐름을 보여주는 데 목적이 있습니다.

```text
출시작 플레이/스토어 분석
  -> 핵심 루프와 개선 여지 도출
  -> 추가 시스템 가설 수립
  -> AI를 활용한 빠른 프로토타입 구현
  -> 직접 플레이하며 재미와 문제점 검증
  -> 기획 문서와 개선 과제로 정리
```

분석의 기준은 다음과 같습니다.

- 기존 Spin Miner의 핵심 재미는 회전 곡괭이, 채굴, 재화 획득, 업그레이드, 탐험 진행에서 나온다고 판단했습니다.
- 채굴 대상이 반복될수록 플레이가 단순해질 수 있기 때문에, 대형 광물에 **지연 노출되는 약점 공략과 브레이크 단계**를 추가해 순간적인 판단과 손맛을 강화하는 방향을 가정했습니다.
- 하이브리드 캐주얼 장르 특성상 조작은 단순하게 유지하고, 피드백과 보상 체감은 즉각적으로 보여주는 것을 우선했습니다.

## 지원 포지션과의 연결점

이 프로젝트는 게임 기획자가 AI 도구를 활용해 아이디어를 문서에만 머무르게 하지 않고, 실제 조작 가능한 프로토타입으로 구현해본 사례입니다.

특히 슈퍼센트의 Spin Miner 콘텐츠/시스템 기획 포지션과 관련해 다음 역량을 보여주는 것을 목표로 합니다.

- **라이브 게임 분석**: 출시작의 핵심 루프와 유저 경험을 분석하고 개선 가설로 연결
- **시스템 기획**: 약점, 균열, 브레이크, 카고, 챕터 진행 등 핵심 규칙을 구조화
- **하이브리드 캐주얼 감각**: 짧은 조작, 즉각적인 피드백, 반복 가능한 성장/진행 루프 설계
- **프로토타이핑 역량**: 기획 의도를 코드로 빠르게 검증하고 개선 포인트를 도출
- **AI 활용 역량**: ChatGPT/Claude 계열 도구를 활용해 기획 정리, 코드 구현, 디버깅, 문서화를 병행
- **협업 준비도**: 개발자와 대화 가능한 수준으로 시스템 단위, 변수, 상태, 개선 이슈를 문서화

## 분석 기반 개선 가설

### 가설 1. 채굴 대상에 공략 차이를 주면 반복 피로를 줄일 수 있다

Spin Miner의 기본 재미는 곡괭이가 회전하며 광석을 부수는 직관적인 손맛에 있습니다.  
하지만 채굴 대상이 단순히 HP만 다른 오브젝트로 반복되면, 중후반 플레이에서 행동 선택지가 줄어들 수 있습니다.

그래서 대형 광물은 처음부터 약점을 보여주지 않고, 일정 시간 공격하면 약점이 드러난 뒤 공략할 수 있는 구조를 제안했습니다.

### 가설 2. 대형 광물은 "오래 때리는 대상"보다 "공략해서 깨는 대상"이 더 기억에 남는다

대형 광물이 단순히 HP만 높다면 플레이어는 같은 행동을 오래 반복하게 됩니다.  
반대로 약점, 균열, 브레이크 상태를 넣으면 플레이어가 광물을 관찰하고 접근 위치를 조정하게 됩니다.

이 방식은 조작 난도를 크게 올리지 않으면서도, 하이브리드 캐주얼 게임에 가벼운 액션성과 공략감을 더할 수 있습니다.

### 가설 3. 대형 광물 앞에서도 플레이어의 개입을 유지해야 한다

대형 광물의 문제는 단순히 파괴 시간이 길다는 점보다, 직접 이동하며 채굴하던 플레이어가 갑자기 기다리는 관전자로 바뀔 수 있다는 점이라고 판단했습니다.  
브레이크 시스템은 대형 광물 앞에서도 플레이어가 위치를 잡고 약점을 공략하는 역할을 유지하도록 설계한 장치입니다.

약점을 공략하면 균열이 빠르게 쌓이고, 브레이크 상태에서는 채굴 속도가 급격히 빨라집니다.  
이를 통해 큰 광물을 "오래 때리는 대상"이 아니라, 컨트롤에 대한 보상을 주는 공략 대상으로 바꾸는 것을 목표로 했습니다.

### 가설 4. 카고 귀환 루프는 짧은 세션 목표로 활용할 수 있다

광물을 캐면 먼저 플레이어 개인 카고에 광석이 쌓이고, BASE에 도착해야 스테이지 카고에 적재됩니다.  
이 구조는 채굴, 적재, 귀환, 비우기 흐름을 만들고, 한 스테이지 안에서도 작은 완료감을 반복해서 주기 위한 것입니다.

## 핵심 기획 의도

### 1. 단순 채굴을 "공략"으로 확장

일반 광물은 곡괭이가 닿으면 바로 채굴되지만, 대형 광물은 단순히 오래 때리는 방식만으로는 효율이 낮습니다.  
대형 광물은 일정 시간 공격해야 약점이 드러나고, 드러난 약점을 지속적으로 공격하면 균열이 쌓이며, 균열이 100%가 되면 브레이크 상태로 전환됩니다.

이를 통해 플레이어가 단순 반복 채굴이 아니라 **목표를 보고 접근하는 공략 플레이**를 하도록 설계했습니다.

### 2. 손맛을 만드는 즉각 피드백

채굴이 성공했을 때 파티클, HP 감소, 색상 변화, 화면 흔들림을 넣어 행동 결과가 즉시 보이도록 했습니다.  
특히 대형 광물 브레이크 시에는 노란 파티클과 glow 효과를 사용해 "약점을 공략했다"는 보상을 시각적으로 강조했습니다.

### 3. 짧은 플레이 루프

광물을 캐면 플레이어 개인 카고에 광석이 쌓이고, 플레이어가 BASE에 도착해야 스테이지 카고가 채워집니다.  
카고가 가득 차면 더 이상 채굴할 수 없기 때문에, 플레이어는 직접 BASE로 이동해 카고를 비운 뒤 다시 채굴을 반복합니다.

기본 루프는 다음과 같습니다.

```text
이동 -> 채굴 -> 개인 카고 적재 -> BASE 이동 -> 카고 비우기 -> 재채굴 -> 챕터 클리어
```

이 구조는 하이브리드 캐주얼 게임에서 중요한 **짧은 세션**, **명확한 목표**, **반복 가능한 성장 구조**의 기초를 검증하기 위한 것입니다.

## 구현된 기능

### 일반 광물

- 다양한 형태와 색상으로 스폰
- 곡괭이 팁 접촉 시 HP 감소
- HP 감소에 따른 어두운 오버레이 표시
- HP 바 표시
- 채굴 완료 시 파티클 폭발과 화면 흔들림 적용

### 대형 광물 브레이크 시스템

- 일반 광물보다 큰 육각형 광물
- 일정 시간 공격 후 광물 표면에 약점 표시
- 약점 공격 시 HP와 균열 게이지가 더 빠르게 감소/증가
- 약점이 아닌 부위를 공격해도 HP는 감소하지만 효율은 낮음
- 약점은 일정 시간마다 위치 변경
- 소형 광물 수를 줄이고 중형/대형/초대형 광물 비중 강화
- 기존 대형 광물 HP 2배 상향
- 챕터마다 더 큰 초대형 광물 1개 등장 시도
- 초대형 광물은 대형 광물 기준 HP 3배로 설계
- 균열 100% 도달 시 브레이크 상태 진입
- 브레이크 상태에서는 일정 시간 동안 몸통 공격 대미지 대폭 증가
- 브레이크 시간이 끝나면 균열이 초기화되고 일반 상태로 복귀

브레이크 시스템 흐름:

```text
약점 노출 -> 약점 공격 -> 균열 증가 -> 브레이크 발동 -> 제한 시간 동안 빠른 파괴 -> 일반 상태 복귀
```

### 카고 게이지

- 광물을 채굴하면 플레이어 개인 카고에 광석 적재
- 개인 카고가 가득 차면 추가 채굴 제한
- 말풍선으로 "베이스로 가 카고를 비우세요!" 안내
- BASE 도착 시 개인 카고가 비워지고 스테이지 카고에 반영

### 챕터 진행

- 모든 광물을 채굴하면 챕터 클리어
- 클리어 화면에서 점수 표시
- 클릭 시 다음 챕터로 진행
- 챕터마다 소형/중형/대형/초대형 광물이 함께 등장
- 챕터가 올라갈수록 광물 수와 대형 광물 크기 증가

## 브레이크 시스템 상세

### 1단계: 약점 노출

- 대형 광물을 약 2초간 공격하면 빨간 약점 표시
- 대형 광물 중심에서 일정 거리 떨어진 위치에 약점 배치
- 약점은 pulse 애니메이션으로 강조
- 광물마다 약점 방향을 다르게 설정해 공략 위치를 다양화

### 2단계: 균열 누적

- 곡괭이 팁이 약점 근처에 닿으면 균열 게이지와 HP가 더 빠르게 감소
- 약점이 아닌 몸통 공격도 HP는 감소하지만 효율은 낮게 적용
- 약점은 일정 시간마다 위치가 바뀌어 계속 움직이며 공략하도록 유도
- 균열이 쌓일수록 광물 위에 균열 선이 선명해짐

### 3단계: 브레이크 상태

- 균열 100% 달성 시 브레이크 발동
- 브레이크는 약 5초 동안 유지
- 브레이크 남은 시간은 별도 게이지로 표시
- 화면 흔들림, 파티클, glow 효과로 성공 피드백 제공
- 이후 몸통 공격 대미지가 크게 증가
- 브레이크 중에는 약점 표시를 숨김
- 시간이 끝나면 균열이 초기화되고 다시 약점을 노출해야 함
- 플레이어가 "제대로 공략했다"는 느낌을 받을 수 있도록 설계

```text
게임 시작
  -> 광물 스폰
  -> 클릭/드래그로 이동

  -> 모든 광물 제거 후 카고를 비우면 챕터 클리어
  -> 다음 챕터 진행
```

## 현재 단계에서 확인한 개선 과제

### 시스템 밸런스

- **균열 게이지 유지 방식**: 현재 한 번 쌓인 균열은 유지됩니다. 일정 시간 약점을 공격하지 않으면 균열이 감소하는 방식이 더 긴장감을 줄 수 있습니다.
- **대형 광물 HP 밸런스**: 약점 공략 전 몸통 대미지가 매우 낮아, 공략 유도는 강하지만 답답하게 느껴질 가능성이 있습니다.
- **개인 카고 용량 밸런스**: 카고 한계치가 너무 낮으면 흐름이 자주 끊기고, 너무 높으면 귀환 루프의 의미가 약해질 수 있습니다.

### 콘텐츠 확장

- 약점 위치가 이동하는 2페이즈 대형 광물
- 광물이 느리게 이동하거나 궤도를 도는 패턴
- 곡괭이 속도, 길이, 범위 업그레이드
- 광물 타입별 카고 분리
- 모바일 터치 조작 지원

### 연출 개선

- 브레이크 발동 시 카메라 줌 인
- 챕터 전환 트랜지션
- 효과음과 BGM 추가
- 채굴 성공/실패 피드백 강화

## 실행 방법

별도 설치나 빌드 과정 없이 브라우저에서 바로 실행할 수 있습니다.

```text
index.html 파일을 Chrome 또는 Edge 브라우저로 드래그 앤 드롭
```

권장 환경:

- Chrome 또는 Edge 최신 버전
- 데스크탑 브라우저
- 마우스 조작

---

## 조작법

| 입력 | 동작 |
| --- | --- |
| WASD | 직접 이동 |
| 마우스 클릭 | 해당 위치로 이동 |
| 마우스 드래그 | 드래그 방향으로 연속 이동 |
| 이동 충돌 | 플레이어는 광물을 덮고 지나갈 수 없음 |
| 자동 동작 | 6개의 곡괭이 회전 및 채굴 |
| BASE 접촉 | 개인 카고 비우기 |
| 클리어 화면 클릭 | 다음 챕터 진행 |

## 포트폴리오에서 강조하고 싶은 점

이 프로젝트는 "기획자가 완성 게임을 혼자 개발했다"는 주장보다, **AI 시대의 게임 기획자가 시스템을 빠르게 실험하고 검증하는 방식**을 보여주는 데 초점을 둡니다.

핵심 메시지는 다음과 같습니다.

- 아이디어를 기획서에만 남기지 않고 플레이 가능한 형태로 검증했다.
- 시스템의 재미 요소와 개선 과제를 직접 플레이하며 도출했다.
- 개발자와 협업할 수 있도록 규칙, 상태, 수치, 개선 방향을 문서화했다.
- AI 도구를 활용해 기획, 구현, 검증, 문서화를 빠르게 반복했다.
