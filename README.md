# Spin Miner - Break System Prototype

> HTML5 Canvas 기반 단일 파일 게임 프로토타입  
> 포트폴리오 목적: 하이브리드 캐주얼 게임의 핵심 시스템을 기획하고, AI를 활용해 플레이 가능한 형태로 빠르게 검증

---

## 프로젝트 한 줄 소개

**Spin Miner**는 자동 회전하는 곡괭이로 광물을 채굴하고, 대형 광물은 약점 공략과 브레이크 시스템을 통해 빠르게 파괴하는 탑다운 액션 채굴 게임 프로토타입입니다.

이 프로젝트는 완성형 게임이 아니라, **채굴 손맛**, **대형 오브젝트 공략**, **카고 귀환 루프**, **챕터 진행 구조**가 실제 플레이에서 어떻게 느껴지는지 확인하기 위한 시스템 검증용 프로토타입입니다.

---

## 지원 포지션과의 연결점

이 프로젝트는 게임 기획자가 AI 도구를 활용해 아이디어를 문서에만 머무르게 하지 않고, 실제 조작 가능한 프로토타입으로 구현해본 사례입니다.

특히 슈퍼센트의 Spin Miner 콘텐츠/시스템 기획 포지션과 관련해 다음 역량을 보여주는 것을 목표로 합니다.

- **시스템 기획**: 약점, 균열, 브레이크, 카고, 챕터 진행 등 핵심 규칙을 구조화
- **하이브리드 캐주얼 감각**: 짧은 조작, 즉각적인 피드백, 반복 가능한 성장/진행 루프 설계
- **프로토타이핑 역량**: 기획 의도를 코드로 빠르게 검증하고 개선 포인트를 도출
- **AI 활용 역량**: ChatGPT/Claude 계열 도구를 활용해 기획 정리, 코드 구현, 디버깅, 문서화를 병행
- **협업 준비도**: 개발자와 대화 가능한 수준으로 시스템 단위, 변수, 상태, 개선 이슈를 문서화

## 핵심 기획 의도

### 1. 단순 채굴을 "공략"으로 확장

일반 광물은 곡괭이가 닿으면 바로 채굴되지만, 대형 광물은 단순히 오래 때리는 방식만으로는 효율이 낮습니다.  
대형 광물에는 약점이 있고, 약점을 지속적으로 공격하면 균열이 쌓이며, 균열이 100%가 되면 브레이크 상태로 전환됩니다.

이를 통해 플레이어가 단순 반복 채굴이 아니라 **목표를 보고 접근하는 공략 플레이**를 하도록 설계했습니다.

### 2. 손맛을 만드는 즉각 피드백

채굴이 성공했을 때 파티클, HP 감소, 색상 변화, 화면 흔들림을 넣어 행동 결과가 즉시 보이도록 했습니다.  
특히 대형 광물 브레이크 시에는 노란 파티클과 glow 효과를 사용해 "약점을 공략했다"는 보상을 시각적으로 강조했습니다.

### 3. 짧은 플레이 루프

광물을 캐면 카고가 차고, 카고가 가득 차면 자동으로 BASE에 귀환합니다.  
귀환 후 다시 채굴을 반복하며, 모든 광물을 제거하면 챕터가 클리어됩니다.

기본 루프는 다음과 같습니다.

```text
이동 -> 채굴 -> 카고 누적 -> BASE 귀환 -> 재채굴 -> 챕터 클리어 -> 다음 챕터
```

이 구조는 하이브리드 캐주얼 게임에서 중요한 **짧은 세션**, **명확한 목표**, **반복 가능한 성장 구조**의 기초를 검증하기 위한 것입니다.

## 구현된 기능

### 일반 광물

- 다양한 형태와 색상으로 스폰
- 곡괭이 팁 접촉 시 HP 감소
- HP 감소에 따른 어두운 오버레이 표시
- HP 바 표시
- 채굴 완료 시 파티클 폭발과 화면 흔들림 적용

### 대형 광물 브레이크 시스템

- 일반 광물보다 큰 육각형 광물
- 광물 표면에 약점 표시
- 약점 공격 시 균열 게이지 증가
- 균열 100% 도달 시 브레이크 상태 진입
- 브레이크 상태에서는 몸통 공격 대미지 대폭 증가

브레이크 시스템 흐름:

```text
약점 발견 -> 약점 공격 -> 균열 증가 -> 브레이크 발동 -> 빠른 파괴
```

### 카고 게이지

- 광물을 채굴하면 카고 수치 증가
- 카고가 100에 도달하면 자동으로 BASE 귀환
- BASE 도착 시 카고 초기화
- 귀환 중에는 별도 상태로 전환되어 플레이 흐름을 분리

### 챕터 진행

- 모든 광물을 채굴하면 챕터 클리어
- 클리어 화면에서 점수 표시
- 클릭 시 다음 챕터로 진행
- 챕터가 올라갈수록 광물 수 증가

---

## 브레이크 시스템 상세

### 1단계: 약점 표시

- 대형 광물 중심에서 일정 거리 떨어진 위치에 빨간 약점 표시
- 약점은 pulse 애니메이션으로 강조
- 광물마다 약점 방향을 다르게 설정해 공략 위치를 다양화

### 2단계: 균열 누적

- 곡괭이 팁이 약점 근처에 닿으면 균열 게이지 증가
- 약점이 아닌 몸통 공격은 매우 낮은 대미지만 적용
- 균열이 쌓일수록 광물 위에 균열 선이 선명해짐

### 3단계: 브레이크 상태

- 균열 100% 달성 시 브레이크 발동
- 화면 흔들림, 파티클, glow 효과로 성공 피드백 제공
- 이후 몸통 공격 대미지가 크게 증가
- 플레이어가 "제대로 공략했다"는 느낌을 받을 수 있도록 설계

## 현재 플레이 흐름

```text
게임 시작
  -> 광물 스폰
  -> 클릭/드래그로 이동
  -> 자동 회전 곡괭이로 채굴
  -> 대형 광물은 약점 공략 후 브레이크
  -> 카고가 가득 차면 BASE 자동 귀환
  -> 모든 광물 제거 시 챕터 클리어
  -> 다음 챕터 진행
```

---

## 현재 단계에서 확인한 개선 과제

### 시스템 밸런스

- **균열 게이지 유지 방식**: 현재 한 번 쌓인 균열은 유지됩니다. 일정 시간 약점을 공격하지 않으면 균열이 감소하는 방식이 더 긴장감을 줄 수 있습니다.
- **대형 광물 HP 밸런스**: 약점 공략 전 몸통 대미지가 매우 낮아, 공략 유도는 강하지만 답답하게 느껴질 가능성이 있습니다.
- **카고 귀환 중 채굴 가능 여부**: 귀환 중 채굴을 막을지, 일부 허용할지에 따라 플레이 템포가 달라질 수 있습니다.

### 콘텐츠 확장

- 약점 위치가 이동하는 2페이즈 대형 광물
- 광물이 느리게 이동하거나 궤도를 도는 패턴
- 곡괭이 속도, 길이, 범위 업그레이드
- 광물 타입별 카고 분리
- 모바일 터치 조작 지원

### 연출 개선

- 브레이크 발동 시 카메라 줌 인
- 챕터 전환 트랜지션
- 효과음과 BGM 추가
- 채굴 성공/실패 피드백 강화

## AI 활용 방식

이 프로젝트는 비개발자인 게임 기획자가 AI를 활용해 기획과 구현을 병행한 사례입니다.

AI를 활용한 주요 작업은 다음과 같습니다.

- 게임 시스템 구조 정리
- HTML5 Canvas 기반 프로토타입 코드 작성
- 충돌 판정, 상태 관리, 이펙트 로직 개선
- README 및 포트폴리오 문서화
- Git/GitHub를 통한 버전 관리 학습 및 적용

AI가 코드를 대신 작성하는 것에 그치지 않고, 기획자가 직접 플레이하며 결과를 판단하고, 개선 방향을 다시 정의하는 방식으로 작업했습니다.

## 포트폴리오에서 강조하고 싶은 점

이 프로젝트는 "기획자가 완성 게임을 혼자 개발했다"는 주장보다, **AI 시대의 게임 기획자가 시스템을 빠르게 실험하고 검증하는 방식**을 보여주는 데 초점을 둡니다.

핵심 메시지는 다음과 같습니다.

- 아이디어를 기획서에만 남기지 않고 플레이 가능한 형태로 검증했다.
- 시스템의 재미 요소와 개선 과제를 직접 플레이하며 도출했다.
- 개발자와 협업할 수 있도록 규칙, 상태, 수치, 개선 방향을 문서화했다.
- AI 도구를 활용해 기획, 구현, 검증, 문서화를 빠르게 반복했다.
